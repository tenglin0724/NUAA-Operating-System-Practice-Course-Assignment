4. pipe 要及时关闭
   - 参考 pipe.c
     * 运行 good()
     * 运行 bad()
     * 比较两者区别
   - 读管道，此时管道中没有数据
     - 如果还有写者
       * read() 阻塞，等写者写数据后，返回非 0
     * 如果没有写者
       - read() 立即返回 0
   - 读管道，返回 0
     * 管道中没有数据
     * 管道没有写者
